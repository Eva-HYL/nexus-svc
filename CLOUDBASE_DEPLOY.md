# Nexus 后端 - 微信云托管部署指南

## 一、准备工作

### 1. 开通云开发
1. 微信开发者工具 → 云开发控制台
2. 创建云开发环境（按量付费或包年包月）

### 2. 开通云托管
1. 云开发控制台 → 云托管 → 开通
2. 选择容器服务

### 3. 创建数据库

**方案 A：云数据库（推荐）**
1. 云开发控制台 → 数据库 → 创建集合
2. 适合轻量应用，免费额度较大

**方案 B：MySQL 版云数据库**
1. 云开发控制台 → 数据库 MySQL → 创建实例
2. 适合需要复杂查询的应用

---

## 二、部署步骤

### 方法一：命令行部署

```bash
# 1. 安装 cloudbase CLI
npm install -g @cloudbase/cli

# 2. 登录
tcb login

# 3. 初始化
tcb init

# 4. 部署
tcb framework deploy
```

### 方法二：控制台部署

1. 云托管控制台 → 新建服务
2. 选择「现有代码」→ 上传代码包
3. 或关联 GitHub/GitLab 仓库自动部署

---

## 三、环境变量配置

在云托管控制台配置：

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `DATABASE_URL` | 数据库连接 | `mysql://root:xxx@host:3306/nexus` |
| `JWT_SECRET` | JWT 密钥 | 随机32位字符串 |
| `WECHAT_APPID` | 小程序 AppID | `wx1234567890` |
| `WECHAT_SECRET` | 小程序 Secret | `abc123...` |

---

## 四、数据库迁移

```bash
# 方式一：使用 Prisma Migrate
npx prisma migrate deploy

# 方式二：云控制台导入
# 1. 导出本地数据库
mysqldump -u root -p nexus > nexus.sql

# 2. 云数据库控制台导入
```

## 五、小程序配置

更新 `elva-miniapp/src/services/api.ts`:

```typescript
const BASE_URL = 'https://your-service-id.service.tcloudbase.com/api';
```

或在云开发控制台获取服务域名。

---

## 六、域名配置

1. 云托管控制台 → 服务设置 → 域名
2. 添加自定义域名（可选）
3. 或使用默认域名

---

## 七、常见问题

### Q: 数据库连接失败？
A: 检查 DATABASE_URL 格式，确保云数据库已创建

### Q: 服务启动失败？
A: 查看云托管日志，检查环境变量配置

### Q: 小程序请求失败？
A: 确保 API 域名已添加到小程序合法域名列表

## 九、监控告警

1. 云托管控制台 → 监控
2. 配置告警规则（CPU、内存、请求量）
3. 绑定微信接收告警通知
